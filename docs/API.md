# Quest Board — API Reference

All API routes are served under the `/api` prefix (proxied by Nginx to the FastAPI backend). Authentication routes live under `/auth`. Bot routes live under `/api/bot/`.

Authentication is via an `HttpOnly` session cookie (`qb_session`) set after a successful OIDC login. All `/api/*` endpoints (except `/api/bot/*`) require this cookie. Bot endpoints require an `X-Bot-Key` header instead.

## Authentication

### `GET /auth/login`

Initiates the OIDC PKCE authorization flow. Redirects the browser to the identity provider.

**Rate limit:** 10 requests/minute per IP.

**Query parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `invite_code` | string | No | Required for first-time registration if `INVITE_CODE` is set in config |

**Response:** `302 Redirect` to the provider's authorization endpoint.

---

### `GET /auth/callback`

Handles the redirect from the OIDC provider after successful authentication.

**Rate limit:** 10 requests/minute per IP.

**Query parameters:**

| Parameter | Type | Description |
|-----------|------|-------------|
| `code` | string | Authorization code from the provider |
| `state` | string | CSRF state token |

**Response:** `302 Redirect` to `/dashboard` with `qb_session` cookie set.

**Error responses:**
- `400` — Invalid or expired state parameter (CSRF protection)
- `403` — Invalid invite code (registration gate)
- `502` — Failed to communicate with OIDC provider

---

### `GET /auth/logout`

Deletes the server-side session and clears the cookie.

**Rate limit:** 10 requests/minute per IP.

**Response:** `302 Redirect` to `/login` with cookie deleted.

---

### `GET /auth/link`

Completes a Discord account linking flow initiated by the bot. The user must be logged in. The one-time token (generated by the bot and stored via `POST /api/bot/linking-tokens`) is consumed, the Discord ID is linked to the current user, and the browser is redirected to `/profile?linked=discord`.

**Rate limit:** 10 requests/minute per IP.

**Query parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `token` | string | Yes | One-time linking token generated by the Discord bot |

**Response:** `302 Redirect` to `{APP_URL}/profile?linked=discord`.

**Error responses:**
- `400` — Token missing, expired (10-minute TTL), or already used
- `409` — The Discord ID in the token is already linked to a different Questboard account

**Response `200`:**

```json
  "display_name": "string",
  "display_name_override": "string | null",
  "effective_display_name": "string",
  "email": "string | null",
  "avatar_url": "string | null",
  "timezone": "string | null",
  "is_admin": false,
  "last_login_at": "datetime | null",
  "created_at": "datetime"
}
```

---

### `PATCH /api/me`

Updates the current user's profile. Only provided fields are changed.

**Request body:**
```json
{
  "display_name_override": "string | null",
  "timezone": "string | null"
}
```

**Response `200`:** Updated `UserResponse`.

## Platform Links

### `GET /api/me/platform-links`

Returns all platform accounts linked to the current user.

**Response `200`:**
```json
[
  {
    "platform": "discord | matrix",
    "platform_user_id": "string",
    "linked_at": "datetime",
    "verified_at": "datetime | null"
  }
]
```

`verified_at` is set when the link was completed via the bot's `/auth/link` flow rather than entered manually.

---

### `POST /api/me/platform-links`

Links (or re-links) a platform account to the current user. If the platform is already linked, the existing entry is replaced.

**Request body:**
```json
{
  "platform": "discord | matrix",
  "platform_user_id": "string"
}
```

**Response `201`:** `PlatformLinkResponse` for the created/updated link.

---

### `DELETE /api/me/platform-links/{platform}`

Unlinks the given platform account from the current user.

**Path parameters:** `platform` — `discord` or `matrix`

**Response `204 No Content`**

**Error responses:**
- `400` — Unknown platform value
- `404` — No link exists for this platform

## Campaigns

### `GET /api/campaigns`

Lists all campaigns the authenticated user belongs to.

**Response `200`:** Array of campaign summaries:
```json
[
  {
    "id": "uuid",
    "name": "string",
    "game_system": "string | null",
    "description": "string | null",
    "created_at": "datetime",
    "my_role": "gm | player"
  }
]
```

---

### `POST /api/campaigns`

Creates a new campaign. The creator is automatically assigned the GM role.

**Request body:**
```json
{
  "name": "string (required)",
  "game_system": "string | null",
  "description": "string | null",
  "discord_webhook_url": "string | null",
  "timezone": "string | null",
  "reminder_offsets_minutes": "[integer] | null",
  "vote_notification_mode": "each_vote | all_voted | null",
  "vote_auto_close_hours": "number | null",
  "guild_id": "string | null",
  "notification_channel_id": "string | null"
}
```

- `discord_webhook_url` must start with `https://discord.com/api/webhooks/` or the request will be rejected with `422`.
- `guild_id` and `notification_channel_id` must be numeric Discord snowflake strings if provided.
- `reminder_offsets_minutes` is an array of integers (minutes before session); up to 3 values.

**Response `201`:** Full `CampaignResponse` (see GET below).

---

### `GET /api/campaigns/{campaign_id}`

Returns full details for a campaign. Requires campaign membership.

**Response `200`:**
```json
{
  "id": "uuid",
  "name": "string",
  "game_system": "string | null",
  "description": "string | null",
  "discord_webhook_url": "string | null",
  "timezone": "string | null",
  "reminder_offsets_minutes": "[integer] | null",
  "vote_notification_mode": "each_vote | all_voted | null",
  "vote_auto_close_hours": "number | null",
  "guild_id": "string | null",
  "notification_channel_id": "string | null",
  "invite_code": "string | null",
  "created_at": "datetime"
}
```

---

### `PATCH /api/campaigns/{campaign_id}`

Updates campaign fields. Requires GM role. Only supplied fields are changed (partial update).

**Request body:** Same fields as `POST /api/campaigns`, all optional.

**Response `200`:** Updated `CampaignResponse`.

---

### `DELETE /api/campaigns/{campaign_id}`

Deletes the campaign and all associated data. Requires GM role.

**Response `204 No Content`**

---

### `POST /api/campaigns/join`

Joins a campaign using its invite code. The joining user becomes a player.

**Request body:**
```json
{ "invite_code": "string" }
```

**Response `200`:** `CampaignResponse` for the joined campaign.

**Error responses:**
- `400` — Invalid invite code or user is already a member

---

### `POST /api/campaigns/{campaign_id}/invite/regenerate`

Generates a new invite code, invalidating the previous one. Requires GM role.

**Response `200`:** Updated `CampaignResponse`.

---

### `GET /api/campaigns/{campaign_id}/next-session`

Returns the next upcoming confirmed session for this campaign, or `null` if none.

**Response `200`:** `SessionListItem` or `null`.

---

### `GET /api/campaigns/{campaign_id}/analytics`

Returns aggregated analytics for a campaign. Accessible to all campaign members.

**Response `200`:**
```json
{
  "total_sessions": 12,
  "proposed_sessions": 1,
  "confirmed_sessions": 2,
  "completed_sessions": 8,
  "cancelled_sessions": 1,
  "average_gap_days": 14.5,
  "sessions_last_30_days": 2,
  "members": [
    {
      "user_id": "uuid",
      "display_name": "string",
      "role": "gm | player",
      "sessions_eligible": 8,
      "sessions_attended": 7,
      "attendance_rate": 0.875,
      "vote_sessions_eligible": 4,
      "vote_sessions_participated": 3,
      "vote_participation_rate": 0.75
    }
  ]
}
```

`attendance_rate` and `vote_participation_rate` are `null` when there are no eligible sessions yet. Only sessions created after a member joined are counted as eligible.

---

### `GET /api/campaigns/{campaign_id}/my-notes`

Returns the current user's aggregated session journal for this campaign: the user's own notes (any visibility) and public GM notes, ordered chronologically by session time.

**Response `200`:**
```json
[
  {
    "session_id": "uuid",
    "session_title": "string | null",
    "confirmed_time": "datetime | null",
    "notes": [
      {
        "id": "uuid",
```

## Sessions

### `GET /api/campaigns/{campaign_id}/sessions`

Lists all sessions for a campaign. Requires campaign membership.

**Response `200`:** Array of `SessionListItem` objects:
```json
[
  {
    "id": "uuid",
    "campaign_id": "uuid",
    "title": "string | null",
    "scheduling_mode": "vote | direct | tentative",
    "status": "proposed | confirmed | completed | cancelled",
    "confirmed_time": "datetime | null",
    "created_at": "datetime"
  }
]
```

---

### `POST /api/campaigns/{campaign_id}/sessions`

Creates a new session. Requires GM role.

**Request body:**
```json
{
  "title": "string | null",
  "description": "string | null",
  "scheduling_mode": "vote | direct | tentative",
  "proposed_times": ["datetime"]
}
```

**`proposed_times` constraints:**
- `vote` mode: 2–5 datetime values
- `direct` or `tentative` mode: exactly 1 datetime value

For `direct` mode, the session is immediately `confirmed` and Discord notifications are scheduled. For `vote` mode on a bot-connected campaign, the bot is notified to post a vote message.

**Response `201`:** Full `SessionResponse` object.

---

### `GET /api/sessions/{session_id}`

Returns full session details including time slots and transcript data. Requires campaign membership.

**Response `200`:**
```json
{
  "id": "uuid",
  "campaign_id": "uuid",
  "title": "string | null",
  "description": "string | null",
  "scheduling_mode": "vote | direct | tentative",
  "status": "proposed | confirmed | completed | cancelled",
  "confirmed_time": "datetime | null",
  "recording_url": "string | null",
  "transcript": "string | null",
  "summary": "string | null",
  "transcript_updated_at": "datetime | null",
  "created_by": "uuid",
  "created_at": "datetime",
  "time_slots": [
    { "id": "uuid", "proposed_time": "datetime", "created_at": "datetime" }
  ]
}
```

---

### `PATCH /api/sessions/{session_id}`

Updates editable session fields. Requires GM role.

**Request body (all optional):**
```json
{
  "title": "string | null",
  "description": "string | null",
  "proposed_times": ["datetime"],
  "reschedule_time": "datetime | null"
}
```

- `proposed_times` replaces all slots on a non-confirmed session.
- `reschedule_time` moves a confirmed session to a new time and reschedules all reminder tasks.

**Response `200`:** Updated `SessionResponse`.

**Error responses:**
- `400` — Using `proposed_times` on a confirmed session, or `reschedule_time` on a non-confirmed session

---

### `DELETE /api/sessions/{session_id}`

Cancels a session (sets status to `cancelled`). Requires GM role. Pending reminder tasks are revoked. On bot-connected campaigns, the bot is notified.

**Response `204 No Content`**

**Error responses:**
- `400` — Session is already cancelled

---

### `POST /api/sessions/{session_id}/confirm`

Confirms a proposed session. Requires GM role. Schedules Discord reminder notifications.

**Request body:**
```json
{ "time_slot_id": "uuid | null" }
```

- **Vote mode:** `time_slot_id` is required (the winning slot).
- **Tentative mode:** omit `time_slot_id`; the single slot is used automatically.

**Response `200`:** Updated `SessionResponse` with `status=confirmed` and `confirmed_time` set.

**Error responses:**
- `400` — Session is not in `proposed` state, slot not found, or vote mode missing `time_slot_id`

---

### `GET /api/sessions/{session_id}/calendar.ics`

Downloads an iCalendar (`.ics`) file for a confirmed session. Requires campaign membership.

The event spans 4 hours from the confirmed time (no actual end-time data is stored). Import into any calendar application or use the `webcal://` URL for Apple Calendar subscription.

**Response `200`:** `text/calendar` file download.

**Error responses:**
- `400` — Session is not confirmed

## Session Notes

### `GET /api/sessions/{session_id}/my-notes`

Returns all notes the current user has written for this session.

**Response `200`:**
```json
[
  {
    "id": "uuid",
    "session_id": "uuid",
    "user_id": "uuid",
    "content": "string",
    "visibility": "private | public",
    "updated_at": "datetime"
  }
]
```

---

### `PUT /api/sessions/{session_id}/my-note`

Creates or updates the current user's note for this session. Sending again replaces the previous note.

Players can only save `private` notes. GMs may set `visibility: "public"` to share notes with all campaign members in the journal view; if a non-GM sends `public`, it is silently stored as `private`.

**Request body:**
```json
{
  "content": "string",
  "visibility": "private | public"
}
```

**Response `200`:** Updated `SessionNoteResponse`.

## Bot API

All bot endpoints require an `X-Bot-Key` header containing the shared secret configured in Admin → Bot Settings. These endpoints are intended for use by the Discord bot service, not by the browser client.

### `GET /api/bot/sessions/upcoming`

Returns confirmed sessions in the next 7 days across all campaigns.

**Response `200`:**
```json
[
  {
    "session_id": "uuid",
    "campaign_id": "uuid",
    "campaign_name": "string",
    "title": "string | null",
    "confirmed_time": "datetime",
    "webhook_url": "string | null"
  }
]
```

---

### `GET /api/bot/sessions/{session_id}/timeslots`

Returns time slots for a vote-mode session with per-slot vote tallies and campaign metadata.

**Response `200`:**
```json
{
  "session_id": "uuid",
  "campaign_name": "string",
  "guild_id": "string | null",
  "notification_channel_id": "string | null",
  "slots": [
    {
      "slot_id": "uuid",
      "proposed_time": "datetime",
      "votes": { "yes": 2, "maybe": 1, "no": 0 }
    }
  ]
}
```

---

### `PUT /api/bot/sessions/{session_id}/timeslots/{slot_id}/vote`

Casts or updates a vote on behalf of a Discord user. The Discord user is resolved to a Questboard user via their linked platform account.

**Request body:**
```json
{
  "discord_user_id": "string",
  "availability": "yes | maybe | no"
}
```

**Response `200`:** `VoteResponse`.

**Error responses:**
- `404` — Discord user has no linked Questboard account, or slot not found

---

### `PUT /api/bot/sessions/{session_id}/attendance/{discord_user_id}`

Marks attendance for a Discord user. The Discord user is resolved via their linked platform account.

**Error responses:**

### `GET /api/bot/link-status/{token}`

Polls whether a Discord account linking flow has completed. The token is consumed on the first `true` response.

**Response `200`:**
```json
{ "linked": false }
```

or, when complete:

```json
{ "linked": true, "user_id": "uuid" }
```

### `GET /api/bot/settings`

Returns the Whisper transcription and LLM summarisation configuration for the bot.

**Response `200`:**
```json
{
  "whisper_endpoint_url": "string",
  "whisper_api_key": "string",
  "llm_endpoint_url": "string",
  "llm_api_key": "string",
  "llm_model": "string"
}
```

---

### `POST /api/bot/linking-tokens`

Stores a one-time Discord account linking token generated by the bot. The user then visits `GET /auth/link?token=<token>` to complete the flow.

**Request body:**
```json
{
  "token": "string",
  "discord_user_id": "string"
}
```

**Response `201`:** `{ "detail": "ok" }`

## Error Responses

All error responses follow the FastAPI default format:

```json
{ "detail": "Human-readable error message" }
```

Validation errors (422) include a `detail` array:

```json
{
  "detail": [
    {
      "type": "value_error",
      "loc": ["body", "field_name"],
      "msg": "Error message",
      "input": "submitted_value"
    }
  ]
}
```

| HTTP Status | Meaning |
|-------------|---------|
| `400` | Bad request — business rule violation |
| `401` | Not authenticated — missing or expired session cookie |
| `403` | Forbidden — insufficient role (not a member / not GM / not admin) |
| `404` | Resource not found |
| `409` | Conflict — e.g. Discord ID already linked to another account |
| `422` | Validation error — request body fails schema validation |
| `429` | Rate limit exceeded |
| `500` | Internal server error (details not exposed) |
| `502` | Bad gateway — upstream OIDC provider error |
